Lesson 0

Before we begin. Setup instructions.

Before we begin, we have to install Haxe - Specifically, the Haxe compiler and the Haxe standard library:

  • The Haxe compiler is a program that will take any code we will write, and convert it into something our computer can run. (There is a bit more to it than that, but we don't have to worry about that for now.)
  • The Haxe standard library is a collection of code that does common tasks for us, so we don't have to write code to do those tasks ourselves - for example, the library provides mathematical functions like sine and cosine.

To intall all of this, go to http://haxe.org/download/ and download the binary for the platform you're on from the top of the page. Run the installer you downloaded and accept all the defaults given.

Now, open a terminal on your computer.
On windows, press the Windows Key and R, then type cmd, which brings up the text terminal.
On Mac, press cmd and space to bring up spotlight, then type terminal to open a terminal window.
Once you got the window open and navigate to the game folder that comes with this tutorial. If you're not familiar with how to navigate using the terminal, here's a quick explanation: On the left side of your cursor, you will see the current directory you are in displayed, for example C:\Windows or Your-Username:~.

Use the cd command to change directory, but typing "cd path/to/go/to" (without the quotation marks), where "path/to/go/to" is the path to a folder you want to navigate to. For example, if you were in your documents folder, and had a folder named "games" in it, you would use "cd games" to move into that folder. To check what files and folders exist in the folder you are currently in, you can use the dir command on windows, or the ls command on Mac.

Once you have managed to the game folder, run the command "haxe build.hxml". This command will tell the haxe compiler to execute the instructions written in the file "build.hxml", which we have configured to compile the game code you will be writing into the game. After running this command, you should see a "build" folder in the game folder, containing a file called "index.html" and a file called "main.js". If you double-click the index.html file, it should open in your browser and display the game as it exists now - A grey background with a green square in the middle. Don't worry though, you'll change that soon!

Once you start changing the code in the following lessons, use the command haxe build.hxml to re-compile your changed code - this means that any changes you have made will take effect in the actual game you can play in your browser. While doing so, the haxe compiler also checks for errors in your code that prevent it from being compiled, so expect this to happen! Luckily however, the compiler will tell you the file and line number in which an error occured, so you can then go in and fix it!


If all of this is working correctly, you can move on to Lesson 1!
