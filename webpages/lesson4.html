Lesson 4

Vectors, velocity, player movement and homing missiles.

So far, we've made the player move around based on keyboard input. Let's take a closer look at how movement is represented in the game, and how the homing missile object follows the player.

Vectors


A vector is a quantity which has magnitude and direction (as opposed to scalars, which only have magnitude). For example, the distance between two locations is a scalar; but the displacement of one location from another i.e. the distance in a certain direction, is a vector.

In this game, vectors are expressed in the Cartesian coordinate system, having an x-component, which represents distance along the horizontal axis; and y-component, which represents distance along the vertical axis. Entities' boxes are moved by adding a vector to their x or y variables, which together represent its position on the plane. Adding of vectors is done component-wise. A vector's length can also be adjusted by multiplying it by a value. Look in the Vector class for the operations on vectors used in this game.

You may have noticed that when moving diagonally, the player object moves faster than when moving only in the horizontal or vertical axes. This is because when two keys are pressed, the player's position is added to by two vectors instead of one. To overcome this, we need to "normalize" the diagonal vector that is created when adding the horizontal and vertical vectors, that is to say, change its length to one so that the player moves at the same rate when moving diagonally.

The class Vector has a property named length. We can set this to change the distance which the player moves each time its state is updated. Consider the equation:

speed = distance/time, and following from that

distance = speed x time

The time taken in this case is the time between each frame. Looking at the function update in the Player class, the reason for passing the variable dt becomes apparent: in order to calculate the distance that the player should move, we need the time between each frame.

Homing missiles


The homing missile is a projectile that constantly follows the player. To achieve this, we need to be able to constantly speed it in a direction towards the player object. This is possible by finding the difference in the player's position and the missile's position, and then adding velocity to the missile in this direction. The difference in the player's position and missile's position can be expressed as a vector, so we can add this vector to the velocity: Vector property of the missile. Missile also has the two properties speed: Float and steering_impulse: Float. speed is meant to be the max speed of the missile, and steering_impulse could control the length of the vector to add to the missile's velocity, in order to adjust the rate of homing.

Exercise 4


In the file Player.hx, find the code that moves the player, which looks like this: 

rect.move(move.x, move.y);
Above it, insert code that sets the length property of the Vector move to the appropriate value based on the player's speed and the time between each frame, to make sure that diagonal movements are at the same speed.


In the file Missile.hx, implement code which makes the missile follow the player.
